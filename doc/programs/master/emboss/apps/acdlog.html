acdlog

Function

Description

Acdlog allows developers to see the detail of ACD file parsing and processing.

Acdlog does not have its own command line. Instead, it will run the command line of any EMBOSS program by reading its ACD file and processing the command line and user responses in exactly the same way as the true program.

Acdlog uses a very simple method. The first command line parameter is the name of the EMBOSS program to be compiled. All other command line parameters and qualifiers are processed by that program's ACD file.

Processing of the ACD file is reported to a file with the same name as the program, and the extension ".acdlog".

Usage

Command line arguments

The first parameter is the name of the program to compile. All other command line parameters and qualifiers are defined for that program - see the documentation for the program of choice.

Output file format

The output file of the program being logged is empty, but any format available to the specified program can be used.

The processing output is written to a file with the extension ".acdlog"

Data files

Acdlog will use any data files specified in the ACD file of the specified program, but will ignore any data files that are only used directly by the program's code.
